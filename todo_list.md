### 📋 话题切换上下文优化方案（已实现）

**核心目标**：保留关键信息、压缩冗余旧话题、隔离无关历史、防止上下文溢出，同时兼顾跨话题必要关联。

**1. 话题意图识别（第一步）** ✅ 已实现
- 通过轻量分类 / LLM Prompt 做话题聚类
- 判定关联性等级：强关联 / 弱关联 / 完全无关

| 关联性 | 场景示例 | 优化策略 | 实现状态 |
|--------|----------|----------|----------|
| **强关联** | 先聊 Python 爬虫，再问爬虫异常报错 | 完整保留历史上下文，不做裁剪 | ✅ |
| **弱关联** | 先聊 Python 爬虫，再问 Python 数据分析 | 保留通用背景，压缩前序细节 | ✅ |
| **完全无关** | 先聊爬虫开发，突然切换成小学生英语练习题 | 触发上下文优化，旧话题摘要归档 | ✅ |

**2. 三种核心优化手段**

| 方案 | 说明 | 实现状态 |
|------|------|----------|
| **滑动上下文窗口** | 设定最大 Token 上限，切换话题后先对旧话题批量摘要压缩（100-300 字结构化摘要），用摘要替换完整对话记录 | ✅ 已实现（50% 滑窗 + 智能边界） |
| **分层记忆架构** | 永久全局记忆（用户偏好）+ 短期话题记忆（当前活跃）+ 归档历史记忆（冷存储向量库） | ⚪ 待实现（需 sqlite-vss） |
| **上下文选择性过滤** | 系统 Prompt 约束，让模型主动区分有效上下文和无关噪声 | ⚪ 待实现 |

**3. 技术实现要点** ✅
- **LLM 话题分析**：使用 LLM 判断新旧话题关联性，返回 JSON 格式的分析结果
- **关键词启发式回退**：当 LLM 不可用时，使用关键词重叠度进行话题检测
- **智能边界压缩**：在话题切换点立即触发压缩，而非等待 50% 水位触发
- **归档检索机制**：旧话题压缩存入向量库，新话题语义命中时重新加载（待实现）

**4. 极端场景处理**（待实现）
- 用户主动说明「换个话题，前面不用参考」→ 旧话题全部归档，只保留系统人设
- 上下文即将 Token 溢出 + 新旧话题完全无关 → 直接丢弃旧话题详细对话，仅保留全局偏好

**5. 设计取舍原则**
- ❌ 不会完全清空历史（防止切回旧话题需重复描述需求）
- ✅ 不主动遗忘用户全局约束（如「回答简洁」「代码带注释」）
- ✅ 弱关联适度复用（同领域保留背景，跨领域强裁剪）
