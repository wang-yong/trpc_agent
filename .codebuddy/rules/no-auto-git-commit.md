# 不自动提交 Git 规则

## 规则说明

在 trpc_agent 项目开发过程中，**禁止**在每次修改或完成功能时主动或自动进行 git commit/push 提交。

## 具体要求

1. **不要主动提交**：除非用户在对话中单独、显式、明确要求提交 git 归档，否则不要执行任何 git commit 或 git push 操作

2. **功能完成后的处理**：
   - ✅ 更新 `todo_list.md` 中的任务状态（将对应的 `[ ]` 标记为 `[x]`）
   - ✅ 向用户说明功能已完成
   - ❌ 不要自动执行 git add、git commit、git push

3. **例外情况**：
   - 用户明确说"提交 git"、"commit"、"归档"等指令时
   - 用户使用 git 相关命令时

## 原因

这个规则确保：
- 用户对代码提交有完全的控制权
- 可以在提交前进行最终检查
- 避免意外提交不完整的功能
- 保持 git 历史的清晰和可控

## 示例

### ❌ 错误做法
```bash
# 完成功能后自动提交
git add .
git commit -m "feat: 完成某功能"
git push
```

### ✅ 正确做法
```markdown
1. 完成代码修改
2. 更新 todo_list.md 标记任务完成
3. 告诉用户："功能已完成，如需提交请告诉我"
4. 等待用户明确要求后再提交
```
