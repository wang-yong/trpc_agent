### 技术栈

- **后端**：Go + trpc-agent-go 框架
- **前端**：Vue 3 + Vite + Naive UI（构建产物 embed 到 Go 二进制，支持单二进制运行部署）
- **LLM**：OpenAI 兼容协议，通过硅基流动（SiliconFlow）接入
- **存储**：内存（两层用户级字典缓存）/ 分用户物理分片 `bin/sessions/` / SQLite（Token 统计）

### 当前架构

```
cmd/server/main.go           → 启动入口，配置多模型、注册 LLMAgent
  ↓
internal/agent/agent.go      → Agent 组装与系统提示词（注册 10 大王牌工具集与执行规范）
  ↓
internal/tools/              → 物理工具包实现
  ├─ calculator/             → 数学四则计算生态（calculator）
  ├─ file/file.go            → 文件系统生态（list_directory, read_file, write_file, edit_file, glob_files）
  ├─ web/web.go              → 网络生态（web_search[百度/DDG自愈], web_scrape, http_request）
  └─ command/command.go      → 终端命令生态（run_command 30s 熔断与安全白名单）
  ↓
internal/server/server.go    → HTTP-SSE 多租户网关（X-User-Id 清洗路由、多路事件追溯、双端 JSON 自愈、首部格式残留剔除、调试日志写入）
  ↓
internal/server/static/      → 前端页面嵌入适配 (static.Dist embed 机制)
  └─ dist/                   → web/ 编译后的单页面应用静态产物 (HTML/CSS/JS)
```

## 二、前端设计规范与完成度

> 现已建立起了一套极致现代感的 AI 设计规范（配色、排版、阴影、微交互），实现了全系统无缝统一。

### 统一设计指标

*   **圆角（Border Radius）**：全局统一收拢为 **`12px`**。
*   **阴影（Shadow）**：三层细粒度外阴影 + 极其微妙的内高光外发光（Lifted Layer），确保弹窗浮空立体感。
*   **动态主题反射（Reactive Theme Mapping）**：`App.vue` 自动监听 Naive UI 主题切换，将所有设计颜色（如 `--border-color`, `--primary-color`, `--card-color` 等）实时反射绑定至根节点 `html`，彻底打通所有 Portal 弹窗、Markdown 及原生 CSS 变量。

## 三、目标分解（5 大自研进阶里程碑 🚀）

根据我们“**不依赖第三方接口、深度自研、聚焦丰富自身智能体特权能力**”的务实核心构想，我们重新整理并设计了以下 5 大自主掌控的进阶里程碑：

### 阶段零：前端重构（基础设施）- **✅ 已完成 (100%)**
> 纯 HTML/JS → Vue 3 + Vite + Naive UI，为 Agent 能力打下最坚实的前端基建，成果超出预期，UI/UX 质感极其出色。

### 阶段一：Agent 推理引擎（可视化）- **✅ 已完成 (100%)**
> 实现 Thought-Action-Observation 闭环并流式呈现，SSE 协议多路追溯，DeepSeek-R1 思考链与 Qwen 自愈 100% 跑通。

### 阶段二：工具生态（能力扩展基础）- **✅ 已完成 (100%)**
> 成功研发并注册了 calculator、list_directory、read_file、write_file、edit_file、glob_files、web_search（百度/DDG自愈）、web_scrape、http_request、run_command 在内的 10 大核心基础工具。

### 阶段三：自研高级开发工具链与安全网 - **🔄 进行中 (已完成 50%)**
> 深度丰富并完善 Agent 的自研生产力特权，打造高安全的人机协同闭环
- [x] **`grep_search`（全局代码正文爆头检索）**：在整个代码仓库的文件正文中，利用标准 regex 或字符串极速搜索匹配文本。在项目巨大、模型需要精确定位某类调用或错误发生时，起到一枪命中的 Grep 作用。
- [x] **`notify_approval`（高危操作拦截与人机审批防线）**：当 Agent 决定调用 `run_command`、`write_file`、`edit_file` 等高危敏感工具时，本地 Go 程序会挂起 SSE 思考流并向前端发出确认信号，前端弹出高光审批框，必须由用户亲手点击“允许”方可物理执行。
- [ ] **`git_commit_helper`（Git 极简变更协作大师）**：一键自动读取物理 Git diff、分析并撰写 Conventional Commits 格式的完美 commit 提交记录，让 Agent 拥有自发进行版本控制归档的协作能力。
- [ ] **`integrated_browser`（自研无头浏览器自动化）**：引入 Go 的 ChromeDP 或者是 Playwright 驱动。允许 Agent 自主打开、导航、截图、点击网页，自动进行复杂网站的数据爬取、自动登录及网页端自动化测试。

### 阶段四：长期记忆、本地 RAG 与知识库构建 - **(规划中)**
> 让 Agent 拥有记忆和私有知识，可以越用越聪明
- [ ] **短期记忆压缩**：会话上下文窗口智能滑动管理 + 超过窗口限制后自动通过 LLM 摘要压缩。
- [ ] **长期记忆检索**：利用本地轻量向量库（如 pgvector/sqlite-vss 等）检索历史对话与用户偏好，建立用户专属画像。
- [ ] **私有知识库挂载**：支持上传本地 PDF/Markdown 知识文档，全自动进行 PDF 切片、Embedding 并实现本地 RAG 引用。

### 阶段五：多 Agent 自主协作生态（终极目标） - **(规划中)**
> 让 Agent 拥有团队，能够解决极为复杂的系统性开发任务
- [ ] **多 Agent 角色定义**：系统规划者 + 代码执行者 + 单元测试审查者。
- [ ] **多 Agent 自主协作**：任务自主拆解分发、内部通信通道、协作看板与不同头像气泡渲染。
