```
┌────────────────────────────────────────────────────────┐
│               最上层：Vue 3 现代化浏览器前端            │

                           ▼
┌────────────────────────────────────────────────────────┐
│               中继层：Go HTTP-SSE 智能网关              │

└──────────────────────────▲─────────────────────────────┘

                           ▼
┌────────────────────────────────────────────────────────┐

```

## 🎨 二、 前端 UI/UX 功能清单 (Frontend Capabilites)

前端基于 **`Vue 3 + Vite + Naive UI + Pinia`** 精心重构，拥有极致精致的设计语言与微交互反馈：

### 1. 极致双主题系统 (Dual-Theme Engine)
*   **极致亮色主题 (Sophisticated Light)**：以科技靛蓝色 `#4f46e5` 作为品牌主色，辅以干净温润的背景 `#fafafa` 及高对比度锌灰边框 `#e4e4e7`。
*   **黑曜蓝调暗色主题 (Obsidian Dark)**：以深邃蓝黑 `#0f111a` 作为主调，卡片采用稍亮的 `#151724`，弹出层采用最亮 `#1e2132` 创造绝对立体的浮空深度（Lifted Layer）。
*   **全域变量反射**：`App.vue` 实时监听主题变化，将色彩同步映射到根节点 `html`，使全局 Markdown、代码高亮以及最外层挂载的 Portal 下拉菜单色调**绝对一致，彻底杜绝半透明刺眼、融在背景里的毛刺体验**。

### 2. 流式 ThinkingChain 思考链卡片 (`ThinkingChain.vue`)
*   **可视化 Timeline 步骤条**：大模型的每一次“起心动念”被渲染为 Timeline，并具有流式发光指示圈（Thinking / Executing / Success）。
*   **双向参数与观测透视**：点击步骤卡片可直接查看 AI 投递工具的 JSON 输入参数（Arguments）以及物理执行结束后返回的绿色代码块观测结果（Observation）。
*   **状态汇总抬头**：根据当前步骤自动归纳展示“正在执行工具: calculator”或“已调度并完成 N 个工具调用”。
*   **响应式折叠**：一键折叠或展开，极简留白。

### 3. 会话状态与导航体验
*   **智能双向导航**：User 弹出菜单在主界面时提供“Token 统计”选项；一旦点击进入统计页面，该项**自动且智能地变更为“返回对话”**。
*   **Logo 一键回首页**：侧栏 Logo 栏支持交互状态与 `pointer` 手势，点击秒级回到对话主界面。
*   **外侧自隐藏交互**：User 弹出窗口具有全局点击侦听，选择完毕或点击外侧任何区域时，**弹窗自动收起并消失**，不给用户添堵。

## ⚡ 三、 后端服务与 SSE 智能路由层 (Go Backend & SSE Router)

后端使用 **`Go 1.24`** 强力驱动，配合 `trpc-agent-go` 建立起全双工流式分发通道：

### 1. 多路复用 SSE 事件解析路由 (`server.go`)
重构了流式通信协议，对大模型单次请求多次 ReAct 迭代事件进行了极细粒度的数据解码，向下分发五大专属 SSE 事件流：
1.  **`thought`**：流式推送大模型思考流（Reasoning Content），原生适配 DeepSeek-R1 / Hunyuan 推理思考过程展现。
2.  **`tool_call`**：流式推送模型请求调用工具的结构体。
3.  **`observation`**：流式推送物理工具运行结束后产生的实际结果 JSON。
4.  **`delta`**：推送常规的总结回答文本流。
5.  **`done`**：会话正常完结信号。

### 2. 完美的流式工具参数追溯
*   针对大模型流式传输中“仅首包带有 ID/Name，后继 Arguments 碎片包 ID 剥离为空”的标准缺陷，后端引入了 **`indexToID` 与 `indexToName` 动态映射表**。
*   根据流式序列号 `tc.Index` 追溯还原所有参数碎片的真实 Tool ID，保证前端 SSE 推送的数据 100% 格式完整、精确匹配！

## 🛡️ 四、 自愈与安全防御网 (Self-Correction & Safety Boundary)

系统加装了多达四层的高级容错与防越权保护，具备工业级的抗波动性：

### 1. 损坏大括号自愈器 (JSON Healing)
*   **故障源**：第三方大模型服务商（如硅基流动）对 Qwen/DeepSeek 的工具 arguments JSON 串打包时常会出现双大括号损坏故障（输出如 `{{ "a": 125 ... }}` 等不合法 JSON）。
*   **前后端联防自愈**：后端与前端 `ThinkingChain.vue` 双向挂载了**括号剥离切片器**。检测到此不合法特征时，瞬时从物理上将其切掉、重整为规范的单大括号 JSON `{ ... }`，**100% 防御反解析报错**。

### 2. 流式前缀剪枝器 (Pristine Delta Trimmer)
*   **故障源**：服务商网关常会将上一回合工具结束时遗留的 `}`、`},`、`]` 等流碎末混在下一回合的 Content 文本流开头输出，污染主界面。
*   **解决**：在 delta 推送前挂载了无状态的 **`isDanglingJsonChunk` 剪枝器**，一旦发现首包碎片纯由 JSON 边界符和换行组成，直接将其剔除过滤，只将绝无杂质的纯净自然语言最终答案发送给用户。

### 3. 安全沙箱边界校验 (`ensureSafePath`)
*   所有涉及本地文件系统操作的工具，强行通过路径安全校验器进行路径标准化 (`filepath.Clean`)，**前缀强行锁死在当前项目工作区根目录下**。
*   任何大模型胡乱生成的 `../../etc/passwd` 等逃逸越权路径，直接触发**安全沙箱拦截**，保护宿主机不被越权访问。

### 4. 迭代熔断安全网 (Infinite Loop Protection)
*   为 LLMAgent 强制注入原生参数：
    *   **`MaxToolIterations = 5`**：单轮任务最多连续调用 5 次工具，超时强制阻断并通知用户。
    *   **`MaxLLMCalls = 10`**：防止大模型陷入逻辑死循环消耗空用户的 API 额度。
